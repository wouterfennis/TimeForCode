# Arc42 Section 12 — Glossary

Status: Current

This glossary defines the terms used throughout the TimeForCode architecture documentation and codebase. Terms are listed alphabetically.

---

| Term | Definition |
| --- | --- |
| Access Token | A short-lived JWT issued by the Authorization API. Used by the Website and downstream services to authenticate API calls. Expires in 60 minutes by default. |
| ADR | Architecture Decision Record. A document that captures the context, decision, and consequences of a significant architectural choice. |
| Arc42 | A lightweight, practical template for software architecture documentation. See [arc42.org](https://arc42.org). |
| Authorization API | The internal service responsible for OAuth 2.0 login flows, JWT issuance, and token refresh. The only service that signs tokens. |
| Blazor | Microsoft's framework for building interactive web UIs using C# and .NET instead of JavaScript. |
| Bounded Context | A DDD concept: a self-contained domain area with its own model, language, and storage. TimeForCode has Authorization, Donation, and Website contexts. |
| Callback | The redirect target that GitHub calls after the user authorises the OAuth 2.0 login. The Authorization API processes this callback to exchange the authorization code for a token. |
| Contributor | A developer who logs hours against an active donation. Can belong to a DonorOrganization or act independently. |
| CQRS | Command Query Responsibility Segregation. A pattern that separates read (query) and write (command) operations. TimeForCode uses a lite variant via MediatR. |
| DDD | Domain-Driven Design. An approach to software development that centres on the business domain, its language, and its rules. |
| Donation | A pledge of a specific number of developer hours from a donor to a project. Has a lifecycle: Pending → Active → Completed or Cancelled. |
| DonationTransaction | A single time-log entry made by a contributor against an active donation. Records hours, a description, and an optional link to a GitHub commit or pull request. |
| DonorOrganization | A company registered on the platform that pledges developer hours on behalf of its employees. |
| GitHub Entity | A base class in the Donation domain that links a domain object to a GitHub resource (repository, user, milestone). |
| HttpOnly Cookie | A browser cookie that cannot be accessed by JavaScript. Used to store tokens securely and prevent XSS attacks. |
| Identity Provider (IdP) | An external service that authenticates users. In TimeForCode, GitHub is the primary IdP. |
| Identity Provider Mock | A test service that simulates the GitHub OAuth 2.0 provider for local development and integration testing. |
| Internal JWT | A JSON Web Token issued by the Authorization API, signed with the platform's RSA key. Used for all internal service-to-service authorization. Distinct from any token provided by GitHub. |
| JWKS | JSON Web Key Set. A set of public keys used to verify JWT signatures. The Authorization API exposes this at `/.well-known/jwks`. |
| Maintainer | A developer who owns and manages a registered open-source project on the platform. |
| MediatR | A .NET library that implements the mediator pattern for in-process message passing. Used for command dispatch in the Application layer. |
| Milestone | A phase or goal within a project, synced from GitHub milestones. Can be targeted by a donation. |
| MongoDB | The NoSQL document database used for persistence in all TimeForCode services. |
| OAuth 2.0 | An open standard for authorization. Used for GitHub login: the user authorises TimeForCode to read their GitHub profile. |
| Project | A registered open-source software project seeking donated developer hours. Linked to a GitHub repository. |
| Refresh Token | A long-lived token (7 days) used to obtain a new access token without requiring the user to log in again. Rotated on each use. |
| RS256 | RSA Signature with SHA-256. The JWT signing algorithm used by the Authorization API. |
| SonarCloud | A cloud-based code quality and security analysis service. Runs on every pull request as part of the CI pipeline. |
| State (OAuth) | A random value generated by the Authorization API before redirecting to GitHub. Returned in the callback and verified to prevent CSRF attacks. |
| Tech Debt | Known implementation shortcomings that reduce maintainability, security, or reliability and should be addressed in future work. See [Section 11](11-risks-and-technical-debt.md). |

See also: [docs/reference/glossary.md](../../reference/glossary.md) for non-architecture terms.
